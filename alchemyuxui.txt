# 1. UX north star

**The product should feel like it is guiding the user inward, not pushing the user forward.**

That means the UX must do four things in order:

1. reduce noise
2. create recognition
3. build trust
4. invite action

Right now, the brand already has emotional depth and strong substance, but the current experience is heavier on long-form reading than on guided flow, and the three offerings are clear individually but not yet orchestrated as one intentional journey. ([Alchemy Unbounded][1])

# 2. Full UX journey

## Stage 1: Arrival

**User emotion:** curious, tired, quietly misaligned
**User question:** “Does this actually understand what I’m feeling?”
**UX goal:** recognition before explanation
**UI must show:** one sharp headline, one calm visual, one obvious next move
**Avoid:** long hero paragraphs, multiple CTAs, service cards too early

### UI elements

* full-height hero
* one editorial headline
* one 2-line supporting copy block
* one primary CTA
* one secondary text link
* minimal nav

### Laws

**Hick’s Law:** reduce choice at the top to one main action and one softer exploration path. Too many options in the first screen slows emotional commitment.
**Jakob’s Law:** keep nav familiar even if the storytelling is cinematic. Unusual layout is fine, but navigation should still feel instantly understandable.

---

## Stage 2: Recognition

**User emotion:** “This feels true”
**User question:** “Am I broken, or just disconnected?”
**UX goal:** name the internal tension clearly
**UI must show:** short, scrollable recognition scenes with pauses
**Avoid:** jumping into credentials or packages too soon

### UI elements

* text-led sections
* wide whitespace between scenes
* quiet transitions
* no image clutter
* one pull-quote style emotional truth

### Laws

**Miller’s Law:** chunk the emotional narrative into 3 to 5 small beats, not one giant block.
**Gestalt Law of Proximity:** each emotional idea should sit in its own visual cluster so the user can process one truth at a time.

---

## Stage 3: Trust

**User emotion:** cautiously open
**User question:** “Why should I trust Harish?”
**UX goal:** make the guide feel human, not elevated
**UI must show:** lived story, select credentials, emotional honesty, proof
**Avoid:** resume wall, certification overload, guru energy

The current About page already gives strong raw material here: Harish’s self-description, mission, years of full-time coaching since 2021, global client base, and extensive training under Eric Maisel, Jill Badonsky, Leon VanderPol, Stevie Kent, and movement practice rooted in Fighting Monkey. ([Alchemy Unbounded][2])

### UI elements

* split-layout founder section
* timeline or scene-based journey section
* “teachers who shaped the work” strip instead of certificate dump
* testimonial excerpts embedded after founder story

### Laws

**Authority bias, used carefully:** selected credentials help, but should come after emotional trust, not before it.
**Peak-End Rule:** the trust phase should end with a strong emotional truth, not a dry credentials list.

---

## Stage 4: Exploration

**User emotion:** interested, comparing internally
**User question:** “Which path is right for me?”
**UX goal:** help self-selection without overwhelm
**UI must show:** three clear pathways

The current site already has the three pillars:

* Transformational Life Coaching
* Creativity Coaching
* Movement Arts Training. ([Alchemy Unbounded][1])

### UI elements

* three offering panels
* “who this is for”
* “what changes”
* one quiet CTA per offering
* optional compare row at bottom

### Laws

**Hick’s Law:** only three high-level choices is good. Keep it there. Do not add sub-options too early.
**Fitts’s Law:** each offering panel should be large, obvious, and easy to tap on mobile.
**Gestalt Similarity:** all three paths should share structure so the user compares quickly.

---

## Stage 5: Clarity

**User emotion:** “I can imagine what this journey feels like”
**User question:** “What happens if I begin?”
**UX goal:** reduce uncertainty
**UI must show:** simple stage-based process

### UI elements

* vertical journey path
* 5 stages max
* stage name + one-line meaning + optional reveal
* connected visual thread

### Laws

**Miller’s Law:** keep the journey to 5 stages max. More than that feels complex.
**Goal-gradient effect:** people move more easily when they can see the path ahead.

---

## Stage 6: Decision

**User emotion:** ready, but still cautious
**User question:** “Is there pressure?”
**UX goal:** make the CTA feel safe
**UI must show:** discovery call as conversation, not commitment

The current site already uses “Book a Free Discovery Call” and direct contact routes. ([Alchemy Unbounded][1])

### UI elements

* single CTA section
* what happens next
* contact alternatives
* reassurance copy
* short form or calendar embed

### Laws

**Fitts’s Law:** CTA button must be prominent, thumb-friendly, and repeated only after trust is built.
**Peak-End Rule:** the final page must leave the user with calm clarity, not urgency pressure.

---

## Stage 7: Post-action

**User emotion:** relief
**User question:** “Did I do the right thing?”
**UX goal:** reinforce trust after submission
**UI must show:** warm confirmation, next step, response expectation

### UI elements

* thank-you screen
* founder note
* expected response time
* optional journal prompt or reflection line

### Laws

**Zeigarnik effect:** after action, tell the user what happens next so the open loop feels held, not anxious.

# 4. Scroll narrative strategy

Most websites feel like “scrolling paper” because nothing changes emotionally. Every section has the same density, same rhythm, same emphasis, same pace.

This one should move like this:

**pause → discomfort → recognition → trust → possibility → invitation**

So the homepage scroll rhythm should be:

1. opening frame
2. pause
3. quiet discomfort
4. villain: conditioned life
5. shift
6. guide
7. three paths
8. proof
9. invitation

That structure matches the current brand’s themes of deep transformation, coaching, creative expression, and embodied practice, while making the scroll feel directed instead of text-heavy. ([Alchemy Unbounded][1])

# 5. UX laws applied to your product

## Hick’s Law

**Problem:** too many choices too early create hesitation.
**Plan:** one primary CTA in hero, one soft secondary action, then offerings later.

## Fitts’s Law

**Problem:** small or distant CTAs create friction, especially on mobile.
**Plan:** large touch targets, sticky mobile CTA only after trust sections, generous tap areas for each offering.

## Jakob’s Law

**Problem:** if everything feels unfamiliar, users work too hard.
**Plan:** keep storytelling cinematic, but keep nav, forms, and page structure familiar.

## Miller’s Law

**Problem:** too much copy in one block overwhelms processing.
**Plan:** chunk long philosophy into smaller scenes, especially on About and service pages.

## Gestalt Principles

**Problem:** unrelated items feel confusing when visually similar.
**Plan:** group copy, proof, and CTAs with strong spacing logic. Use whitespace as structure.

## Peak-End Rule

**Problem:** users remember the emotional peak and the ending.
**Plan:** strongest emotional truth on homepage and strongest calm invitation on Begin page.

# 7. UI decision system

## Use more whitespace when:

* emotional truth needs to land
* user needs to pause
* you want premium calmness

## Use more density when:

* naming the problem
* comparing offerings
* showing process detail

## Use contrast when:

* shifting emotional act
* moving from confusion to clarity
* highlighting final CTA

## Reduce visual intensity when:

* the copy is already emotionally strong
* trust is being built
* form completion is required

# 8. UI elements to use, and when

## Hero

Use for emotional interruption
Do not overload with buttons or badges

## Split founder block

Use for About and trust scenes
Do not use repeatedly

## Editorial text section

Use for philosophy and recognition
Do not let it run too long without visual or spacing break

## Offering panels

Use for the 3 paths
Do not add icons unless they truly add meaning

## Journey timeline

Use on Journey page
Do not use on homepage

## Witness story block

Use for testimonials
Do not use a carousel

## Quiet CTA section

Use late in page
Do not use as every section ending

## Threshold form

Use only on Begin
Do not scatter forms across the whole product

# 9. Interaction design system

* hover: slight underline shift or tone change
* button press: subtle darken/lighten, no bounce
* section reveal: 250–400ms soft fade + rise
* nav: minimal sticky header after first viewport
* form feedback: warm, clear, human
* loading states: skeletons or soft fades, no flashy spinners

The UI should feel alive, but calm. Never static in a dead way. Never animated in a showy way.

# 10. Motion system

## Use motion for

* directing attention
* signaling scene change
* softening scroll transitions
* drawing the journey thread

## Do not use motion for

* decoration
* proving modernity
* constant parallax
* testimonial autoplay
* over-animated text blocks

## Timing

* fast micro interactions: 120–180ms
* standard hover/focus: 180–240ms
* section reveal: 260–420ms
* bigger transitions: 320–480ms

Stillness should be strongest in:

* hero pause
* founder truth section
* final invitation

# 11. Visual hierarchy system

## Level 1

headline truth
largest type
strongest contrast

## Level 2

supporting explanation
medium width
more breathing room

## Level 3

details like “who this is for” or process steps

## Level 4

captions, labels, metadata

Hierarchy should come from:

* type scale
* vertical spacing
* contrast
* container width

Not from visual noise.

# 12. Cognitive load fixes

These are the likely friction points to avoid:

* too much emotional copy in one block
* too many similarly weighted sections
* service details before trust
* repeating the same emotional promise with different words
* too many CTA placements

Fix with:

* chunking
* fewer equal-weight moments
* stronger transitions
* page-specific jobs
* tighter copy
